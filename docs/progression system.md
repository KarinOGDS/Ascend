# Progression System
### Version 1.0

---

# Philosophy

Ascend is not a workout tracker.

Ascend is a progression system.

Every workout must generate visible progress.

The user should always feel that they earned something after training.

Progress is earned.

Never given.

---

# Core Progression

Ascend has two progression systems.

## 1. Player Progression

Represents the player.

Rewards consistency.

Unlocks avatar evolution.

Unlocks global ranks.

Displayed in the Hero section.

---

## 2. Muscle Progression

Represents each individual muscle.

Every muscle has its own:

- XP
- Level
- Rank
- Rewards
- Visual evolution

Every muscle evolves independently.

---

# Design Rules

Every rank changes:

Frame

Border

Glow

Background

Badge

Effects

The HTML never changes.

Only CSS classes change.

Example:

rank--novice

↓

rank--apprentice

↓

rank--warrior

↓

rank--champion

↓

rank--legend

---

# Golden Rule

Every workout must produce visible progress.

The user should never finish a workout wondering:

"Did I actually progress?"

Instead they should immediately see:

More XP

A fuller progress bar

A new level

A new rank

A new frame

A new reward

Every session should feel rewarding.
